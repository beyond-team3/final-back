# 도메인 WBS (deal / approval / notification / schedule / statistics)

아래 WBS는 현재 코드 구현 상태와 `fijitlwkr` 작성 커밋(BAC-43, 52, 61, 63, 64, 70, 80, 81, 93, 99, 101, 102, 110 등) 이력을 함께 반영해 정리했다.

| 작업 | 기능상세 | 담당자 | 완료여부 |
|---|---|---|---|
| 영업 딜 조회·요약 API 개발 (Deal) | `DocumentSummary` 기반 통합 문서 목록 조회와 권한 스코프(ADMIN/SALES_REP/CLIENT) 필터를 구현하고, 최신 기능 요약 문서(BAC-94)까지 반영 | 이하경 | 완료 |
| 영업 딜 로그·상태 전이 파이프라인 구축 (Deal) | `DealPipelineFacade`/`DealLogWriteService` 중심으로 문서 상태 전이 검증, 로그 기록, 스냅샷 동기화를 연결하고 BAC-70/102에서 null 안정성·원자성·후처리 보강 | 이하경 | 완료 |
| Deal V2 문맥·KPI 확장 (Deal) | BAC-110 기준 `deal-v2` 조회 계층, 딜 컨텍스트 알림/일정 조회, KPI 집계, snapshot 재계산/재동기화, 문서 생성·재작성 초안을 확장 | 이하경 | 완료 |
| 승인 요청 생성·결정 워크플로우 구현 (Approval) | QUO/CNT 대상 승인 요청 생성, 단계별 승인/반려 결정, 요청/단계 상태 전이와 중복 요청 방지 로직을 구현(BAC-64/81) | 이하경 | 완료 |
| 승인 권한·정합성 검증 강화 (Approval) | SALES_REP 소유권 검증, CLIENT 스냅샷 정합성, 단계 순서 검증, 문서 상태 전이 검증, 승인 로그 stage mismatch 방지 등을 반영(BAC-81/93/102) | 이하경 | 완료 |
| 승인 후속 처리·연계 이벤트 안정화 (Approval) | 승인 완료/반려 시 문서 후속 전이, 일정 동기화, 주문 승인 확정 이벤트 전환, 승인 요청 자동 생성·취소 연계까지 보정(BAC-93/99/102) | 이하경 | 완료 |
| 알림 도메인 기본 인프라 구축 (Notification) | 알림 엔티티/저장소, 조회 API, SSE 푸시, 배치 전달 워커/스케줄러, 재배 적기 전용 알림 서비스 기반을 구현(BAC-52) | 이하경 | 완료 |
| 문서 라이프사이클 알림 연동 (Notification) | 견적요청 생성·승인 요청·반려·완료·계약 예약·계정 활성화·상품 등록 등 이벤트 기반 알림을 문서 흐름에 연결(BAC-102) | 이하경 | 완료 |
| 딜 문맥 알림·운영 안정화 (Notification) | 딜 컨텍스트 알림 조회 경로와 재배 적기 운영 알림 경로를 추가하고, visible 경계 및 SSE 종료 로그 노이즈를 개선(BAC-110/102) | 이하경 | 완료 |
| 개인 일정 CRUD 및 통합 조회 구현 (Schedule) | 개인 일정 생성/수정/삭제, 기간 겹침 조회, 통합 일정 응답 모델을 구축하고 validation 규칙을 정립(BAC-61) | 이하경 | 완료 |
| 딜 일정 동기화 파이프라인 구현 (Schedule) | 승인/문서 이벤트 기반 deal 일정 upsert·취소 동기화, assignee fallback, 외부키 충돌 회피 등 운영 결함을 수정(BUG-3/BAC-99) | 이하경 | 완료 |
| Schedule V2 soft-cancel 및 컨텍스트 연계 (Schedule) | BAC-110에서 일정 soft-cancel 처리와 딜 컨텍스트 조회 연계를 추가해 문서/딜 상태와 일정 표시 일관성을 강화 | 이하경 | 완료 |
| 청구 매출 통계 집계 API 구현 (Statistics) | QueryDSL 기반 billing revenue 집계와 기간별(월/분기) 응답을 구현하고 category 필터/과대 집계 이슈를 수정(BAC-63/80) | 이하경 | 완료 |
| 공통 통계 API 확장 (Statistics) | `/api/v1/statistics` 하위에 영업사원/관리자/거래처/품종/랭킹 조회 API를 추가하고 빈 구간 0 보정·권한 제한을 구현(BAC-101) | 이하경 | 완료 |
| 딜 KPI 통계 연계 고도화 (Statistics) | BAC-110에서 deal KPI 조회를 추가해 성공률·전환율·리드타임·재작성률 등 딜 성과 지표를 통계/컨텍스트와 연결 | 이하경 | 완료 |
