프론트 반영용 프롬프트

아래 조건에 맞춰 프론트엔드 코드를 수정하라.

목표
- 기존 `v1` 화면을 유지하면서, 이번 리모델링으로 추가된 `v2` API를 프론트에서 선택적으로 테스트할 수 있게 만든다.
- 절대 기존 `v1` API 경로를 통째로 `v2`로 바꿔치기하지 않는다.
- 전역 `base_url=/api/v1` 구조를 깨지 말고, 필요한 API 함수만 개별적으로 `v2` 절대 경로를 사용하게 한다.
- 환경변수 또는 feature flag로 `v2` 사용 여부를 켜고 끌 수 있게 한다.

핵심 원칙
1. `v1` 기본 동작은 유지한다.
2. `deal`, `quotation`, `contract`, `deal KPI`, `billing revenue`만 `v2` 테스트 대상으로 분리한다.
3. `RFQ`, `ORD`, `STMT`, `INV`, `PAY`, 일반 통계는 기존 `v1` 유지한다.
4. 화면 단위 전환보다 API client 함수 단위 전환을 우선한다.
5. 응답 shape가 달라진 경우 화면 adapter/mapper를 추가해서 흡수하고, 컴포넌트를 크게 갈아엎지 않는다.

반영 대상 API
- `GET /api/v2/deals`
- `GET /api/v2/deals/{dealId}`
- `GET /api/v2/deals/{dealId}/documents`
- `GET /api/v2/deals/{dealId}/notifications`
- `GET /api/v2/deals/{dealId}/schedules`
- `GET /api/v2/deals/kpis`
- `POST /api/v2/quotations`
- `POST /api/v2/quotations/{quotationId}/revise`
- `PATCH /api/v2/quotations/{quotationId}/cancel`
- `POST /api/v2/contracts`
- `POST /api/v2/contracts/{contractId}/revise`
- `PATCH /api/v2/contracts/{contractId}/cancel`
- `GET /api/v2/statistics/billing/revenue/monthly`
- `GET /api/v2/statistics/billing/revenue/by-category`
- `GET /api/v2/statistics/billing/revenue/monthly-by-category`

권장 구현 방식
1. feature flag 추가
- 예시:
  - `VITE_USE_DEAL_V2=true`
  - `VITE_USE_QUOTATION_V2=true`
  - `VITE_USE_CONTRACT_V2=true`
  - `VITE_USE_BILLING_STATS_V2=true`
- 또는 하나로 묶어서:
  - `VITE_ENABLE_PIPELINE_V2=true`

2. API client 분기 추가
- 기존 공통 `baseUrl`은 그대로 둔다.
- 아래 함수만 개별적으로 `/api/v2/...` 절대 경로 사용:
  - deal 목록/상세/문서/알림/일정/KPI
  - quotation 생성/재작성/취소
  - contract 생성/재작성/취소
  - billing revenue 통계

3. mapper 추가
- `v2` 응답이 기존 `v1` 화면 모델과 다르면 adapter를 만든다.
- 예:
  - `mapDealV2SummaryToViewModel`
  - `mapDealV2DetailToViewModel`
  - `mapDealV2DocumentsToTimelineItems`
- UI 컴포넌트는 가능한 그대로 두고 mapper에서 맞춘다.

4. 화면 노출 방식
- `v2` 테스트용 토글이 켜졌을 때만 새 API 사용
- 토글이 꺼져 있으면 기존 `v1` 호출 유지
- 가능하면 화면 어딘가에 `V2 TEST` 배지나 dev-only label을 표시

주의사항
- 기존 `v1` api 함수명을 `v2`로 덮어쓰지 말 것
- 전역 `axios instance`의 기본 prefix를 `/api/v2`로 바꾸지 말 것
- `v1` 화면 전체를 한 번에 전환하지 말 것
- 미구현 도메인까지 억지로 `v2`로 연결하지 말 것
- 백엔드 alias나 임시 rewrite를 전제로 구현하지 말 것

우선 수정 추천 영역
1. deal 메인 목록 화면
2. deal 상세 화면
3. quotation 작성/재작성/취소
4. contract 작성/재작성/취소
5. deal KPI
6. billing revenue 통계
